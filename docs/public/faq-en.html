Frequently asked questions

This page keeps the public boundary clear and answers the questions users are most likely to ask first.

Why is there no download page yet?

The public site is focused on product information, feedback entry, and release notes first. That keeps the brand message clean and the release boundary explicit.

How should feedback be submitted?

Use the Feedback page to open a GitHub Issue. Include the version, environment, steps to reproduce, and any supporting screenshots or logs.

Is there a roadmap?

Only public-facing items are shown. Internal implementation details and specific delivery dates are not published.
